# send-sms-hook Edge Function

Intercepts Supabase Auth SMS sends and delivers OTPs via MSG91 instead of the configured phone provider (Twilio placeholder). Called server-to-server by Supabase Auth — no CORS, no JWT verification.

## How it works

1. Supabase Auth triggers this hook when it needs to send an OTP SMS
2. The hook verifies the webhook signature (HMAC-SHA256 via Standard Webhooks spec)
3. Extracts `user.phone` and `sms.otp` from the payload
4. Calls MSG91 Flow API to deliver the OTP via the registered DLT template
5. Returns 200 on success

If `MSG91_AUTH_KEY` is not set, the hook returns 200 silently (placeholder mode — allows the hook to be deployed and registered in the Supabase dashboard before credentials arrive, without breaking the auth flow).

## Deploy

```bash
supabase functions deploy send-sms-hook --no-verify-jwt
```

The `--no-verify-jwt` flag is required because this hook is called by Supabase Auth before a user session exists.

## Required secrets

Set these before going live:

```bash
supabase secrets set MSG91_AUTH_KEY="your-msg91-auth-key"
supabase secrets set MSG91_FLOW_ID="your-msg91-flow-id"
supabase secrets set MSG91_SENDER_ID="YOURSI"
supabase secrets set SEND_SMS_HOOK_SECRET="v1,whsec_your-secret-from-dashboard"
```

`SUPABASE_URL` and `SUPABASE_SERVICE_ROLE_KEY` are auto-injected — do not set these manually.

### Where to find these values

| Secret | Where to get it |
|--------|----------------|
| `MSG91_AUTH_KEY` | MSG91 dashboard → API Keys |
| `MSG91_FLOW_ID` | MSG91 dashboard → SMS → Flows → your flow ID |
| `MSG91_SENDER_ID` | MSG91 dashboard → Sender ID (must be DLT-approved, exactly 6 chars) |
| `SEND_SMS_HOOK_SECRET` | Generated by Supabase dashboard when you register the hook (step 3 below) |

## Dashboard configuration

### Step 1 — Configure phone provider (Twilio placeholder)

1. Supabase Dashboard → **Authentication → Providers → Phone**
2. Enable **Phone** provider
3. Select **Twilio** as provider
4. Enter placeholder credentials (the hook intercepts before Twilio is called):
   - Account SID: `ACplaceholder00000000000000000000000`
   - Auth Token: `placeholder`
   - Message Service SID / From number: `+15005550006` (Twilio test number)
5. **Disable** "Enable phone confirmations" if you want OTP-less login (password-only)
   — or enable it if you want OTP verification on signup
6. Save

### Step 2 — Deploy the hook

```bash
supabase functions deploy send-sms-hook --no-verify-jwt
```

### Step 3 — Register the hook in Supabase

1. Supabase Dashboard → **Authentication → Hooks**
2. Click **Add hook** → **Send SMS**
3. Set URL to:
   ```
   https://puvyqwrqbvqrovogavev.supabase.co/functions/v1/send-sms-hook
   ```
4. Copy the generated **Secret** value
5. Run: `supabase secrets set SEND_SMS_HOOK_SECRET="v1,whsec_<paste-secret-here>"`

### Step 4 — Set MSG91 credentials

```bash
supabase secrets set MSG91_AUTH_KEY="..."
supabase secrets set MSG91_FLOW_ID="..."
supabase secrets set MSG91_SENDER_ID="..."
```

## MSG91 Flow API — payload format

```json
POST https://control.msg91.com/api/v5/flow/
Headers: { "authkey": "...", "Content-Type": "application/json" }

{
  "flow_id": "your-flow-id",
  "sender": "YOURSI",
  "recipients": [{ "mobiles": "919XXXXXXXXX", "VAR1": "123456" }]
}
```

`VAR1` maps to the OTP variable in your MSG91 DLT-approved template. If your template uses a different variable name, update `index.ts` accordingly.

## Switching to SMSIndiaHub

To replace MSG91 with SMSIndiaHub:
1. Update `MSG91_AUTH_KEY`, `MSG91_FLOW_ID`, `MSG91_SENDER_ID` secrets with SMSIndiaHub equivalents
2. In `index.ts`, replace the MSG91 `fetch()` call with the SMSIndiaHub API call
3. Redeploy: `supabase functions deploy send-sms-hook --no-verify-jwt`
